Objetivo

La motivación principal de este proyecto fue resolver limitaciones significativas de recursos durante el desarrollo de aplicaciones empresariales a gran escala. Los IDEs tradicionales de JetBrains, aunque potentes, integran numerosas utilidades que consumen memoria RAM de forma excesiva, dificultando a menudo el rendimiento en hardware con limitaciones específicas. Tras encontrar el ecosistema de Visual Studio Code poco adecuado para mi flujo de trabajo, pivoté hacia Neovim. El objetivo fue construir un entorno de desarrollo "ligero" que eliminara el exceso de carga (bloat) y recuperara la memoria del sistema para la ejecución y pruebas reales de aplicaciones pesadas.
Filosofía Técnica

Al migrar a una configuración impulsada por Lua, adopté un enfoque "modular-first". A diferencia de los IDEs estándar que cargan todo al iniciar, esta configuración utiliza técnicas de lazy-loading (carga diferida) para garantizar que los plugins y los Servidores de Lenguaje (LSP) solo se inicialicen cuando se abren tipos de archivos específicos. Esto asegura la menor huella de memoria base posible sin sacrificar la potencia de un entorno de desarrollo moderno.
Desafíos Clave

El principal reto técnico consistió en ajustar el Protocolo de Servidor de Lenguaje (LSP) para casos de uso complejos. Mientras que la implementación del autocompletado y el formateo estándar fue sencilla, lograr un soporte de alta calidad para plantillas integradas de Angular y CSS embebido presentó una curva de aprendizaje pronunciada. Configurar el editor para reconocer y proporcionar intellisense dentro de estos contextos anidados requirió profundizar en los analizadores de Treesitter y configuraciones avanzadas de LSP. Curiosamente, esta limitación se convirtió en una ventaja profesional; al gestionar estas secciones manualmente, mejoré significativamente mi memorización de sintaxis y mi comprensión interna de las estructuras de los frameworks.
Resultados

La iteración actual ha transformado con éxito mi experiencia de desarrollo en un flujo de trabajo más rápido y centrado en el teclado, utilizando solo una fracción de la RAM de un IDE tradicional. Aunque el proyecto es una configuración "viva", ya ha estabilizado el rendimiento de mi sistema bajo condiciones de alta carga. Actualmente, estoy explorando plugins avanzados y scripts personalizados en Lua para mejorar aún más las herramientas contextuales y acercar la experiencia a la paridad con las funciones de los IDEs premium.
