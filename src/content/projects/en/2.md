üí° Technical Decisions & Patterns
1. Backend: Hexagonal Architecture (Go)

The Go service implements a Clean Architecture to decouple business logic from infrastructure concerns.

    Repository Pattern: Abstracted DynamoDB access, making the system easier to test and database-agnostic for future migrations.

    Performance: Leveraged Go's concurrency and efficiency to ensure ultra-low latency for the API.

2.Frontend: Docker Optimization

To optimize the deployment on ECS Fargate, the Next.js image uses multi-stage builds and output: standalone. This significantly reduced image size, resulting in faster deployment cycles and more efficient horizontal scaling.
3.Infrastructure as Code (CDK)

Instead of manual console configuration, the entire environment is defined in TypeScript via AWS CDK.

    Atomic Deployments: One command (cdk deploy --all) handles the entire stack.

    Environment Replication: Easily spin up staging or production environments with parity.

    Security: Follows the Principle of Least Privilege (PoLP) for all IAM roles.
