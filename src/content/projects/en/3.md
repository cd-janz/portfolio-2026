    The Goal: Achieve a perfect 100/100 Lighthouse score while supporting a scalable, type-safe multi-language architecture.

1. Robust Internationalization (i18n)

Instead of simple JSON translation files, I implemented a Content Collection-based i18n system.

    Schema Validation: Using Zod, I ensured that every localized project or blog post follows a strict structure, preventing build errors when a translation is missing.

    Dynamic Language Switching: Leveraged Astro‚Äôs new routing logic to handle locale detection and persistent language preferences without page flickers.

2. Performance & View Transitions

I replaced traditional page loads with the Astro View Transitions API.

    The Result: Smooth, animated transitions between pages that preserve state (like a playing video or a sidebar scroll position) without the overhead of a full React/Next.js client-side router.

    Asset Optimization: Utilized the astro:assets image service to automate WebP conversion and responsive sizing, drastically reducing Largest Contentful Paint (LCP) times.
