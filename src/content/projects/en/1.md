üéØ The Mission: Reclaiming Resources

Modern enterprise development often faces a "resource tax." Traditional JetBrains IDEs, while feature-rich, consume significant RAM and CPU cycles on background utilities that aren't always necessary.

    The Problem: Hardware bottlenecks when running heavy microservices alongside a resource-intensive IDE.

    The Pivot: Migrating from the VS Code/JetBrains ecosystem to a fully customized Neovim setup.

    The Goal: A "zero-bloat" environment where every megabyte of RAM is reserved for the execution and testing of the application, not the editor itself.

üí° Technical Challenges & Solutions
1. The "Nested Context" Problem (LSP Tuning)

Standard Language Server Protocol (LSP) setups struggle with multi-language files. Implementing robust IntelliSense for Angular templates and embedded CSS required a deep dive into Treesitter parsers.

    Solution: Custom configuration of nvim-lspconfig and null-ls to bridge the gap between HTML and CSS/TypeScript contexts, ensuring seamless code completion in complex frontend structures.

2. Cognitive Performance vs. Automation

By stripping away "magic" IDE buttons, I encountered a steep learning curve that ultimately became a professional asset.

    The Result: Manually managing complex framework structures improved my syntax memorization and internal understanding of architectural patterns, leading to cleaner code and faster troubleshooting.
